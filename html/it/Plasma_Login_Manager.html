Plasma Login fornisce un Display manager per KDE Plasma, derivato da SDDM e con un nuovo front-end che offre un greeter, l'integrazione dei plugin per lo sfondo e un modulo delle Impostazioni di Sistema (KCM).

Installazione

Installare il pacchetto plasma-login-manager.

Seguire la guida Display manager#Loading the display manager per avviare Plasma Login Manager all'avvio del sistema.

Suggerimento

Se si sta migrando da SDDM, ricordarsi di disabilitare prima sddm.service.

Dopo aver completato la migrazione, è inoltre possibile pulire ed eliminare l'utente sddm, che non sarà più utilizzato, con il comando userdel -r sddm eseguito come root.

Configurazione

Dopo l'installazione, una nuova sezione chiamata Schermata di accesso viene aggiunta alle Impostazioni di Sistema, permettendone la configurazione grafica.

Per modificarlo manualmente, creare un file di configurazione in /etc/plasmalogin.conf. In alternativa inserire i file di configurazione in /etc/plasmalogin.conf.d/.

Plugin per lo sfondo personalizzato

Il modulo per le impostazioni di sistema (KCM) dedicato a Plasma Login Manager potrebbe non riuscire a elencare i plugin per lo sfondo personalizzato o di terze parti, anche nel caso in cui siano correttamente installati in /usr/share/plasma/wallpapers/.

In questo caso è necessario indicare manualmente, nel file di configurazione, l'ID del plugin. L'ID del plugin può essere prelevato dal nome della cartella in /usr/share/plasma/wallpapers/ o dal campo Id nel file metadata.json del plugin. Modificare il file di configurazione nel seguente modo: 

/etc/plasmalogin.conf
[Greeter]
WallpaperPluginId=id.del.proprio.plugin

Salvare il file e uscire dalla sessione per verificare i cambiamenti.

Accesso automatico

Plasma Login supporta l'accesso automatico tramite il suo file di configurazione, ad esempio: 

/etc/plasmalogin.conf.d/autologin.conf
[Autologin]
User=john
Session=plasma.desktop

Questa configurazione avvia una sessione KDE Plasma per l'utente john all'avvio del sistema. I tipi di sessione disponibili si trovano in /usr/share/xsessions/ per X e in /usr/share/wayland-sessions/ per Wayland.

Per accedere automaticamente a KDE Plasma e bloccare simultaneamente la sessione (ad esempio per consentire l'avvio delle app in background), consultare KDE#Schermata di blocco.

Accesso senza password

È possibile configurare Plasma Login per consentire l'accesso ad alcuni account senza password. È diverso dall'accesso automatico poiché l'utente deve comunque scegliere l'account con cui loggarsi e differisce dal semplice impostare una password vuota perché permette l'accesso solo agli utenti interattivi (e non, ad esempio, agli utenti remoti via SSH).

Plasma Login usa PAM, quindi è necessario creare il file di configurazione /etc/pam.d/plasmalogin per PAM: 

/etc/pam.d/plasmalogin
#%PAM-1.0
auth       sufficient   pam_succeed_if.so user ingroup nopasswdlogin
auth       include      system-login
account    include      system-login
session    include      system-login
password   include      system-login

Per consentire anche lo sblocco senza password della schermata di blocco di KDE Plasma, creare anche il file di configurazione /etc/pam.d/kde per PAM: 

/etc/pam.d/kde
#%PAM-1.0
auth       sufficient   pam_succeed_if.so user ingroup nopasswdlogin
auth       include      system-login
account    include      system-login
session    include      system-login
password   include      system-login

È quindi necessario far parte del gruppo utente nopasswdlogin per poter accedere interattivamente senza inserire la password: 

# groupadd -r nopasswdlogin
# gpasswd -a nomeutente nopasswdlogin

Icona dell'utente (avatar)

Per opzione predefinita Plasma Login Manager accede alle icone degli utenti da /var/lib/AccountsService/icons/. Dal menu delle Impostazioni di sistema di Plasma > Utenti si possono definire gli avatar oppure è possibile copiare il proprio file .face.icon (se esiste) nella directory: 

# cp /home/user/.face.icon /var/lib/AccountsService/icons/user
Suggerimento Se si sta migrando da SDDM e si era dato accesso al proprio avatar nella directory home mediante le ACL dei file, è necessario rimuoverle: 
$ setfacl -x u:sddm ~/.face.icon
$ setfacl -x u:sddm ~/
