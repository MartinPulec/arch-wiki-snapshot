Les Mainteneurs de paquets (précédemment appelé Trusted Users soit « Utilisateurs de confiance ») est un rôle officiel de l'équipe d'Arch Linux. Les mainteneurs de paquets remplissent les tâches suivantes :

  • Maintenir les paquets dans les dépôts officiels (à part le dépôt core).
  • Maintenir, gérer et surveiller le fonctionnement de l'AUR.

Le terme générique « mainteneur de paquet » est également utilisé pour décrire toute personne qui maintient un paquet, quel que soit le dépôt, comme décrit dans Arch terminology#Package maintainer.

Comment puis-je devenir un Mainteneur de Paquets ?

Les exigences minimales pour devenir un Mainteneur de Paquets sont les suivantes :

  • connaître les bases des scripts shell
  • maintenir quelques paquets dans l'AUR avec des PKGBUILDs propres et de haute qualité
  • un minimum de participation à la communauté (liste de diffusion, forums, IRC)
  • maîtrise de la recherche sur le web
  • une idée générale du type de paquets que vous voulez maintenir (en fait, pourquoi voulez-vous devenir Mainteneur de Paquets ?).

Même si vous pouvez devenir Mainteneur de Paquets en remplissant simplement ces conditions minimales, les personnes qui vous jugeront lors de la procédure de vote standard pourraient en attendre plus de vous. Par exemple :

  • implication dans le bug tracker (rapport, recherche, information)
  • patches pour les projets Arch
  • participation à quelques projets open-source (même si ce sont les vôtres).

Si vous vous sentez toujours prêt à devenir un Mainteneur de Paquets après avoir lu ces lignes, la première étape est de trouver deux Mainteneur de Paquets qui acceptent de vous parrainer. Une fois parrainé, vous devez écrire une demande pleine d'inspiration signée avec votre clé GPG à la liste de diffusion aur-general.

Note: Si le Mainteneur de Paquets que vous avez contactée refuse de parrainer votre candidature, vous devez le faire savoir si vous cherchez un parrainage auprès d'un autre Mainteneur de Paquets.

Pour plus d'informations, voir le Règlement des Mainteneur de Paquets (en) et Directives pour les Mainteneur de Paquets (en).
