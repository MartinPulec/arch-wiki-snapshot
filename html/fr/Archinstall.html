archinstall est une bibliothèque logicielle qui vise à simplifier l'installation d'Arch Linux. Elles est empaquetée avec plusieurs installateurs pré-configurés tel que l'installateur guidé.

Cette page ne documente pas l'utilisation de archinstall en tant que bibliothèque Python; consultez la documentation officielle[dead link 2024-01-13—HTTP 404] pour cela.

Attention archinstall est expérimental et propose des réglages par défaut différents de ceux d'une installation classique. Pour toute demande de d'aide ou de soutien pour un système installé par ce moyen, pensez à le signaler et à fournir /var/log/archinstall/install.log
Note archinstall prends en charge les systèmes BIOS de façon expérimentale uniquement avec GRUB.

Utiliser l'installateur

Tout d'abord il faut récupérer et démarrer l'environnement live avec la procédure décrite dans la préparation à l'installation. Le paquet archinstall étant inclut dans cette image il peut directement être lancé comme suit: 

# archinstall

L'installateur guidé va suivre les étapes suivantes:

  1. Configuration des locales;
  2. Sélection du miroir;
  3. Partitionnement des disques;
  4. Formatage des partitions;
  5. Mise en place du chiffrement des disques (optionnel);
  6. Définition du nom de l'ordinateur (hostname);
  7. Définition du mot de passe de l'utilisateur root;
  8. Installation d'un chargeur d'amorçage (limité à systemd-boot (en) pour les systèmes UEFI et GRUB pour ceux avec un BIOS).
Attention Laisser un mot de passe vide pour l'utilisateur root désactive celui-ci. Sudo est alors utilisé pour obtenir des privilèges. Cependant aucun compte utilisateur n'est créé par archinstall et désactiver la connexion au compte root empêche toute utilisation du système, aucun utilisateur n'étant alors accessible. Voir Sudo#Disable root login.
Note L'installateur peut configurer des interfaces filaires en utilisant systemd-networkd (Français) et il peut aussi copier les configurations faites dans le système live. Ainsi, si une interface sans-fil est configurée via iwctl (en), la configuration est copiée dans le système installé. Les configurations des interfaces filaires sont aussi copiées.

Des paquets supplémentaires peuvent être installés en écrivant leur nom quand cela est demandé: Write additional packages to install.

Une fois l'installation terminée, un texte en vert apparaît, indiquant qu'il est possible de redémarrer. Cela se fait avec: 

# reboot

Profils

archinstall inclut des profils, ce sont des groupes de paquets pré-configurés qui peuvent être installés en plus du système de base.

Attention Les profils sont fournis par les développeurs d'archinstall et ne sont donc pas maintenus par les empaqueteurs. Il est recommandé de vérifier les détails de chaque profil avant de les utiliser.
