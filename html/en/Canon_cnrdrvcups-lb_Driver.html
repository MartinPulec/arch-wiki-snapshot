Canon networked printers often use UFR II backend for printing. cnrdrvcups-lb-binAUR is a package that installs Canon’s Linux printer driver components (CUPS filters/backends) and PPD files for many Canon devices using UFR II, UFRII LT, and (for some models) LIPSLX or CARPS2.

This package is typically used when:

  • driverless printing (IPP Everywhere/AirPrint/Mopria) is unavailable or missing device-specific options, or
  • the device requires Canon’s UFR/LIPS/CARPS PPDs to print correctly.

Installation

Follow CUPS to install and configure the printing stack (including enabling and starting the CUPS service), then install cnrdrvcups-lb-binAUR.

Configuration

Add a printer (CUPS web interface)

Use the CUPS web UI (see CUPS):

  • http://localhost:631 → Administration → Add Printer.
  • When choosing a driver, select the matching Canon model entry. If needed, choose Provide a PPD file and select a PPD from:
    • /usr/share/cups/model/

Add a printer (command line)

Find available device URIs: 

$ lpinfo -v

List installed Canon PPDs: 

$ lpinfo -m | grep -i '^CNRCUPS'

Create a queue using a specific PPD (example): 

# lpadmin -p Canon -E -v socket://PRINTER:9100 -m CNRCUPSLBP664CZK.ppd
# lpoptions -d Canon

Tip

If your device appears in multiple variants (e.g. UFR II vs UFRII LT vs LIPSLX/CARPS2), choose the variant that matches the printer language configured on the device, or the one that successfully prints. When in doubt, try the UFR II PPD first for newer devices.

Troubleshooting

“Filter failed” / nothing prints

  • Verify the device URI (USB vs network backend) and that the printer is reachable.
  • Try a different PPD variant for the same model (UFR II vs UFRII LT vs LIPSLX/CARPS2).
  • Review logs:
    • journalctl -u cups.service
    • /var/log/cups/error_log (increase LogLevel if needed; see CUPS/Troubleshooting).
