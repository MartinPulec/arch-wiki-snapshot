⎜   ↳ Virtual core XTEST pointer              	id=4	[slave  pointer  (2)]

⎣ Virtual core keyboard                   	id=3	[master keyboard (2)]
    ↳ Virtual core XTEST keyboard             	id=5	[slave  keyboard (3)]
    ↳ Power Button                            	id=6	[slave  keyboard (3)]
    ↳ Video Bus                               	id=7	[slave  keyboard (3)]
    ↳ Video Bus                               	id=8	[slave  keyboard (3)]
    ↳ Sleep Button                            	id=9	[slave  keyboard (3)]
    ↳ Asus WMI hotkeys                        	id=12	[slave  keyboard (3)]
    ↳ AT Translated Set 2 keyboard            	id=13	[slave  keyboard (3)]

No touchscreen.

Fn+Brightness combination does not work. Fn+Volume and media control buttons do work. Fn+ScreenOn/ScreenOff works. Fn+AirplaneMode does not work.

On hibernation touchpad becomes unresponsive. Some users report boot some parameters fix it.

Graphics

One can use the binary driver's PRIME function to use either NVIDIA (High drain, high performance, no hardware video acceleration (h264) meaning vdpau is not accessible? [1] ) or Intel (low drain, hardware video acceleration (h264)).

Firmware

User can disable Secure Boot, AES-NI and Intel Virtualization but cannot disable Intel HyperThreating.

There is a choice of compatibility mode, user reports the option potentially becoming grayed out(?).
