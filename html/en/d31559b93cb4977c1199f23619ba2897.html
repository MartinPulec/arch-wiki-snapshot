From ArchWiki
翻譯狀態：此文乃 Arch_Linux 之翻譯。新輯時日：2025-01-21。閱此查英文版本後之變更。

夫 Arch Linux 者，獨立、通用之 x86-64 GNU/Linux 發行版也。

Arch 盡其力滾動更新，供諸程序之至新之穩定版本。初裝之 Arch 乃至簡之系統，其後君可以己之好，裝己所需，而為己誌也。

原則

簡潔

Arch Linux 視所謂簡為：避所不必之增修。其軟件皆供自原開發者（上遊），僅行極微繕之及發行版（下遊）者：

  • 不與上遊所拒之補。絕乎眾 Arch 下遊之補皆得證於上遊，含於下一正式之版。
  • 配置之件同自上遊，僅含發行版必須之更正，如特殊文件系統路徑之變。若裝一軟件包，Arch 無自動功能，如啓其服務。
  • 軟件包常直指其上遊項目，絕少分之。
  • 正統不與圖形化配置，君應持命令行與文本編輯器以修其設置。

現代

Arch 力保軟件於至新穩定之版。倘系統軟件包無損，則新之也竭力。

Arch 據滾動升級之綱，以求永新於畢裝。

Arch 供 GNU/Linux 用戶以諸新者，含 systemd 初始化系統、現代文件系統、LVM2/EVMS、軟件磁盤陣列（軟 RAID）、udev 之與、initcpio（附 mkinitcpio）及至新之內核。

實用

Arch 重實用，避義理之爭。終之設計與決策者，皆斷於開發者之共識也。開發者據技於實，悟言一室，以去政務之因，非受製於世俗。

Arch Linux 之倉，具軟件包及腳本者以千計，可自擇之。此倉供開源、自由之軟件，亦不乏閉源軟件。 實用重於義理。

忠於用戶

眾 Linux 發行版力求友善於用戶；Arch Linux 則平素永忠於用戶。斯發行版之旨，乃迎貢獻者之需，非求致用戶之多也。

Arch 適於以自強為樂者，其不吝時辰，好覽文解惑。

無論何人，皆勸其與會，建功，自報缺陷而助修繕之，供軟件包之補，以至於與核心項目：Arch 開發者皆乃誌士，致公不輟者以為群之一員。 Archers 可自獻軟件包至Arch 用戶之倉，和ArchWiki 之文質，授技術之知於論壇，郵件之列，IRC 。

Arch Linux 今乃全球眾用戶之選，有眾國際社區獻其助力及編纂譯文。

通用

Arch Linux 之通用，於裝，僅予命令行環境：靡刈包無用者之勞煩，而可擇眾利器於正統軟件之倉，自建其系統。今支持 x86-64。( i686 之支持終矣）

Arch 役包管理系統以滾動更新軟件包乃至全系統。Arch 亦有包構建系統，名曰 Arch build system，其狀類 ports，可構建於源碼而裝軟件包，可同步以一令，亦可復構建全系統以一令，據此皆易。Arch 另供 Arch 用戶之倉，有眾 PKGBUILD 腳本焉，獲維護於用戶；乘 makepkg 之利，腳本之編譯至打包者，一氣呵成也。更有甚者：用戶之構建與維護自之軟件源也易。

歷史

Arch 社區壯大之不輟，亦為近年之議論所見證。

Arch 開發者皆無餉酬，亦無以 Arch Linux 謀利之計。欲聞 Arch 開發之詳史，可閱 Wayback Machine 之 Arch 及 Arch Linux 新聞檔案。

始建

加拿大程序員兼吉他師 Judd Vinet 自公元 2001 年初，始建 Arch Linux，於 2002 年 3 月 11 日公行 0.1 版。玆受啟於 Slackware，BSD，PLD Linux 與 CRUX。然是時，玆發行版皆闕包管理之器。故 Vinet 立此發行版以簡潔之道，撰 pacman 以理包之裝定、刪刈與更新。

既而

archstats2002-2011.png玆表明 Arch Linux 社區之逐步壯大。自始之日，乃樹開放、友好及里仁之形象。

ArchWiki 之始行

公元 2005 年 7 月 8 日，基以 MediaWiki 之 ArchWiki 始行。

A. Griffin 時代

公元 2007 歲半，Judd Vinet 輟 Arch 之開發，準平授率於美利堅程序員 Aaron Griffin，即 Phrakture，今猶任 Arch 開發者。

Arch 安裝腳本

公元 2012 年 7 月 15 日之 Arch Linux 介質棄Arch 裝定框架(AIF) 之基於菜單者，更以Arch 安裝腳本 (arch-install-scripts)。

Systemd 時代

公元 2012 至 2013 年間， Arch 替 System V init 以 Systemd。[1][2][3][4]

棄 i686 之支持

開發者及社區見 i686 之用式微，是故 棄 i686 於公元 2017 年 11 月末 。

項目統領之拔舉

公元 2020 年初，Arch 始舉統領以新法，詳見 DeveloperWiki:Project Leader。

Aaron Griffin 辭項目統領，Arch 遂舉 Levente Polyak 為新統領，此皆受錄於[5]。

GitLab 時代

2023 年 5 月，Arch Linux 遷打包之至簡器具於 GitLab。革新於內；外裂 testing 為 core-testing 與 extra-testing，分 staging 為 core-staging 與 extra-staging，並 community 至 extra。見 GitLab 博客以聞其詳。

是年 11 月，舊之缺陷跟蹤器（Flyspray）遷至 GitLab，其協作之功能（issues 與 merge requests）公行。為檔案之故，存一 Flyspry 之靜態副本，以保鏈接（如FS#56716）之效如舊。
