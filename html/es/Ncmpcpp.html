ncmpcpp (Español)

Appearance
From ArchWiki

Esta traducción de Ncmpcpp fue revisada el 2020-02-19. Si existen cambios puede actualizarla o avisar al equipo de traducción.

Artículos relacionados

  • Music Player Daemon (Español)

Ncmpcpp es un cliente de mpd (compatible con mopidy) con una interfaz de usuario muy similar a la de ncmpc, pero que proporciona nuevas características útiles como el soporte de expresiones regulares para búsquedas en la biblioteca, formato de canciones extendido, filtrado de elementos, la posibilidad de ordenar las listas de reproducción y un navegador de archivos local.

Para utilizarlo, un servidor de mpd funcional debe estar presente en el sistema ya que ncmpcpp/mpd trabajan juntos en una relación cliente/servidor.

Instalación

Instale desde los repositorios oficiales.

Configuración básica

La interfaz gráfica de usuario de "ncmpcpp" es altamente personalizable. Edite ~/.ncmpcpp/config a su gusto. Si, después de la instalación, no se ha creado ~/.ncmpcpp/config, simplemente copie el ejemplo de configuración, cambie los permisos y edite como mínimo las tres opciones de configuración siguientes:

  • mpd_host - Should point to the host on which mpd resides, can be "localhost", "127.0.0.1" or "::1" if on the same machine. To connect with a password, write "password@host"
  • mpd_port - The default of mpd should be "6600"
  • mpd_music_dir - The same directory value as specified in "music_directory" in mpd.conf

Como referencia, consulte los siguientes ejemplos:

  • Ejemplo de configuración en /usr/share/doc/ncmpcpp/config.
  • Muestra tu .ncmpcpp/config

Permitir la visualización

Para la visualización, añada unas líneas en /etc/mpd.conf para permitir la generación de datos fast Fourier transform: 

audio_output {
    type                    "fifo"
    name                    "my_fifo"
    path                    "/tmp/mpd.fifo"
    format                  "44100:16:2"
}

Hay que añadir líneas adicionales a ~/.ncmpcpp/config

visualizer_fifo_path = "/tmp/mpd.fifo" 

visualizer_output_name = "my_fifo"
visualizer_sync_interval = "30" 
visualizer_in_stereo = "yes"
visualizer_type = "spectrum"
visualizer_look = "+|"
  • visualizer_sync_interval - Establece el intervalo para sincronizar el visualizador con la salida de audio de mpd. Debe ser mayor de 10 para evitar tratar de sincronizar con demasiada frecuencia, algo que congela la visualización. El valor recomendado es 30, pero puede reducirse si el audio se aleja en apariencia con la visualización.
  • tipo_visualizer - Configure la visualización a un analizador o a un formulario de spectrum/{{ic|ellipse}/wave_filled.
  • visualizer_look - Establece la apariencia del visualizador (el string debe tener exactamente dos caracteres de longitud: el primero es para la onda y el segundo para el espectro de spectrum, wave_filled y ellipse).

Uso básico

Ejecutar ncmpcpp

Ejecute ncmpcpp en una shell: 

$ ncmpcpp

Diferentes vistas

Diferentes puntos de vista

Lista parcial de opiniones dentro de ncmpcpp:

  • 1 - Lista de reproducción actual
  • 2 - Navegador del sistema de archivos
  • 3 - Búsqueda en la base de datos
  • 4 - Biblioteca
  • 5 - Editor de listas de reproducción
  • 6 - Editor de etiquetas (muy poderoso)
  • 7 - Selector de salida
  • 8 - Visualizador de música
  • = - Reloj
  • F1 - Ayuda

Otras claves de UI

  • q - Salir
  • f - Buscar hacia adelante
  • b - Buscar hacia atrás
  • \ - Cambiar entre la visión clásica y la alternativa
  • # - Mostrar la tasa de bits del archivo
  • i - Mostrar información de la canción
  • I - Mostrar información del artista (guardada en ~/.ncmpcpp/artists/ARTIST.txt)
  • L - Mezclar entre las bases de datos líricas disponibles
  • l - Recuperar las letras de la canción actual Mostrar/esconder las letras
  • > - Próxima pista
  • < - Pista anterior
  • p - Reproducir/Pausar
  • + - Aumentar el volumen 2%
  • -} - Disminuir el volumen 2%

Modos de reproducción

Observe el panel de control en la parte superior derecha; se muestra en modo alternativo: 

1:40/5:59 1082 kbps                            ──┤ La Gitana ├──                                       Vol: 98%
[playing]                       Los Planetas - Zona temporalmente autónoma (2017)                     [-z-c--]

Y ahora en el modo clásico: 

─────────────────────────────────────────────────────────────────────────────────────────────────────────[zc]─

Esto corresponde a los modos de reproducción. Ordenados de izquierda a derecha, son:

  • r - modo bucle [r-----] (Repite la lista de reproducción después de llegar al final)
  • z - modo aleatorio [-z----]
  • y - modo uno solo [--s---] (Sólo toca una pista y luego se detiene)
  • R - modo consumido [---c--] (Elimina la pista actual de la lista de reproducción después de llegar al final)
  • x - modo crossfade [----x-]

El "-" final sólo se activa cuando el usuario fuerza una actualización a la base de datos a través de u.

Para hacer bucle de una sola pista, active los modos simple y repetido.

Consejos y trucos

Remapear las claves

Una lista de los enlaces de teclas y sus respectivas funciones está disponible desde el mismo npmpcpp pulsando F1. Los usuarios pueden reasignar cualquiera de las claves por defecto simplemente copiando /usr/share/doc/ncmpcpp/bindings a ~/.ncmpcpp/ y editándolo.

Etiquetas automáticas de nombre de archivo y viceversa

En el Editor de Etiquetas puede seleccionar un directorio con música y luego seleccionar la opción Nombre del archivo en la sección del medio. Esto abre una pequeña ventana con dos opciones: Obtener etiquetas del nombre de archivo y renombrar archivos. Si elige Obtener etiquetas del nombre del archivo, se muestra un popup con dos ventanas. En el lado izquierdo puede introducir un patrón que extrae la información seleccionada de los nombres de archivo. También puede pulsar Previsión para ver cómo sería el resultado. En el lado derecho se puede ver el texto que contiene todas las posibles palabras clave a utilizar para la extracción.

Un ejemplo sencillo sería el patrón: %a - %t. Si sus archivos se nombran de acuerdo con este patrón (Artista - Título), entonces este patrón extraería esta información y establecería las etiquetas para el archivo.

La otra opción Renombrar archivos hace exactamente lo contrario. Toma las Etiquetas de los archivos de audio y crea un nombre de archivo a partir de ellos.
