Traducido desde la pagina del proyecto:

El «Lightweight X11 Desktop Environment» (Entorno de escritorio ligero para X11) es un entorno de escritorio que ahorra energía y funciona extremadamente rápido. Es mantenido por una comunidad internacional de desarrolladores, viene con una interfaz hermosa, provee soporte para múltiples lenguajes, ocupa atajos de teclado estándar y tiene características adicionales como la navegación de archivos a través de pestañas. LXDE ocupa menos CPU y RAM que otros entornos. Esta especialmente diseñado para computadores enfocados en la nube con pocos recursos de hardware, como notebooks, dispositivos móviles (p.ej. MIDs) o computadores más antiguos.

Instalación

LXDE requiere que por lo menos este instalado lxde-common, lxsession y Openbox (u otro gestor de ventanas). El grupo lxde contiene el escritorio completo.

GTK 3 versión

Puede instalar una compilación experimental de LXDE con GTK 3 con el grupo lxde-gtk3.

Aunque funciona mayormente, hay problemas conocidos con gpicview, lxappearance-obconf, lxlauncher y lxpanel.

Iniciando el escritorio

Inicio grafico

LXDM es el gestor de inicio por defecto de LXDE y es instalado como parte del grupo lxde. Vea también Display manager.

Consola

Para usar startx, añada a xinitrc: 

~/.xinitrc
exec startlxde

Vea también Start X at login.

Consejos y trucos

Editando el menú de aplicaciones

El menú de aplicaciones funciona indexando los archivos .desktop ubicados en /usr/share/applications/ y ~/.local/share/applications/. Para añadir o editar un ítem del menú, vea desktop entries. Puede encontrar editores de menús en el AUR (p.ej. lxmedAUR). También hay oficiales, como alacarte (gnome), mozo (mate), etc..

Inicio automático

LXDE Implementa XDG Autostart. Puede iniciar aplicaciones de manera automática de las siguientes maneras:

  • Con los archivos .desktop
  • A través de LXsession

Cada linea en ~/.config/lxsession/LXDE/autostart representa un comando que se va a ejecutar. Si la linea empieza con @, y el comando que la sigue falla, el comando es reejecutado automáticamente. Por ejemplo: 

~/.config/lxsession/LXDE/autostart
@lxterminal
@leafpad
Nota: Estos comando no terminan con un símbolo «&».

También hay un archivo de autoinicio global en /etc/xdg/lxsession/LXDE/autostart.

Nota: Si ambos archivos existen, LXsession solo va a ejecutar el archivo local desde la versión v0.4.9

Atajos

Los atajos de teclado y ratón son implementados con Openbox. Los usuarios de LXDE deben seguir la wiki de Openbox[enlace roto 2024-10-12] (en inglés) para editar ~/.config/openbox/lxde-rc.xml.

Una interfaz grafica opcional para editar los atajos es proveída por el paquete obkeyAUR. Aunque edita rc.xml por defecto, puede hacer que edite el archivo de configuración de LXDE así: 

$ obkey ~/.config/openbox/lxde-rc.xml

Vea [1] para más información.

Cursores

lxappearance es una herramienta grafica para cambiar el aspecto y estilo de GTK, incluyendo el tema del cursor. Los ajustes establecidos con LXAppearance son escritos en ~/.gtkrc-2.0, ~/.config/gtk-3.0/settings.ini y ~/.icons/default/index.theme. Vea también Temas del cursor.

Applet de reloj digital

Puede cambiar la forma en que se muestra la hora al presionar con el botón derecho del ratón y ocupando el formato strftime. Vea strftime(3) para los detalles.

Configuración de las fuentes

lxappearance-obconf establece la configuración de Openbox. Vea Configuración de fuentes.

Distribución del teclado

lxpanel incluye un applet de la distribución del teclado. Vea Keyboard configuration in Xorg para instrucciones genéricas, y #Inicio automático para iniciar setxkbmap automáticamente en LXDE.

Bloqueo de pantalla

LXDE no viene con su propio bloqueador de pantalla. Vea List of applications/Security (Español)#Aplicaciones para bloquear la pantalla e #Inicio automático para ver como iniciarlo.

El icono Screen Lock ejecuta un script (ubicado en /usr/bin/lxlock) que busca entre un numero de bloqueadores muy conocidos y ocupa el primero que encuentra para bloquear la pantalla. Vea lxlock en GitHub.

/etc/xdg/lxsession/LXDE/autostart (de el paquete lxde-common) lista XScreenSaver, el que va a ser ejecutado automáticamente.

Vea DPMS para ver como controlar el salvapantallas sin programas externos.

Iconos de LXPanel

Los iconos que usa LXpanel por defecto están en /usr/share/pixmaps/ y cualquier icono personalizado debe guardarse ahí.

Para camiar los iconos por defecto de las aplicaciones, vea Desktop entries.

Menú de LXPanel

El menú del panel puede ser configurado en /etc/xdg/menus/lxde-applications.menu siguiendo el formato xdg-menu para usarlo con las aplicaciones de otra sesión (notablemente MATE) para añadir alguna de la funcionalidad faltante de LXDE.

Usando un gestor de ventanas diferente

LXsession usa el gestor de ventanas definido en ~/.config/lxsession/LXDE/desktop.conf (por defecto Openbox). si este archivo no existe, busca en /etc/xdg/lxsession/LXDE/desktop.conf.

Reemplace openbox-lxde en cualquiera de los dos archivos con su gestor de ventanas deseado:

Para usar metacity: 

window_manager=metacity

Para usar compiz: 

window_manager=compiz

Alternativamente use WM --replace como se define en #Inicio automático, donde WM e el nombre del gestor de ventanas a ejecutar. Esto significa que openbox va a ser ejecutado en cada inicio de sesión y va a ser reemplazado inmediatamente. Tome nota que Openbox y LXDE no comparten el mismo rc.xml y los atajos de teclado pueden diferir. Vea xbindkeys.

Usando un compositor

LXDE no tiene la composición habilitada por defecto, lo cual puede producir artefactos visuales. Estos pueden ser arreglados al costo de un poco de rendimiento grafico instalando un compositor, como picom.

Solución de problemas

Algunas aplicaciones no inician

Sugerencia: Algunos usuarios reportan que estos problemas ya están resueltos en la ultima actualización de LXDE.

GTK3 LXDE posee algunos problemas que pueden impedir que las aplicaciones se inicien (notablemente lxappearance). Hasta que esto sea arreglado, puede usar JWM para evitar la violación de acceso con gtk3:

  • Abra una terminal y ejecute jwm.
  • Puede que se produzca algún error grafico, pero ahora debería poder ejecutar la aplicación normalmente.
  • Cuando termine, vuelva a iniciar sesión para reiniciar el escritorio.

NTFS con caracteres chinos

Para usar un dispositivo de almacenamiento con un sistema de archivos NTFS, tiene que instalar el paquete NTFS-3G.Generalmente, PCManFM funciona bien con sistemas de archivos NTFS, aunque hay un bug que afecta a los que usan NTFS en el que si se tiene archivos o directorios en un sistema de archivos NTFS, el nombre de los que tienen caracteres no latinos (p.ej. caracteres chinos) puede desaparecer al abrir (o auto-montar) el volumen NTFS. Esto sucede porque el asistente de montado de lxsession no esta interpretando las políticas y opciones de locales correctamente. Puede solucionarlo temporalmente así:

Cree un /usr/local/bin/mount.ntfs-3g nuevo con un script de Bash que contenga: 

#!/bin/bash
/usr/bin/ntfs-3g $1 $2 -o locale=en_US.UTF-8

Y hágalo ejecutable: 

# chmod +x /usr/local/bin/mount.ntfs-3g

LXPanel falla

Con algunos temas para GTK, al iniciar lxpanel se produce el siguiente error: 

lxpanel: cairo-scaled-font.c:459: _cairo_scaled_glyph_page_destroy: Assertion `!scaled_font->cache_frozen' failed.

En este caso, instale ttf-dejavu.

Si lxpanel falla cuando se navegan ciertas paginas con caracteres unicode, instale ttf-droid.

Arreglar el tamaño de los iconos de la barra de tareas LXPanel

Los iconos de las aplicaciones ejecutándose no son del Icon size establecido en Panel Settings > Geometry, si no que son 4px más pequeños lo que hace que algunos estén borrosos. Para que los iconos midan 32px y no se vean borrosos, Icon size debe estar establecido a 36px en la barra de tareas, esto hará que los iconos de los applets en el panel estén borrosos. Para arreglarlo, cree paneles adicionales y ubiquelos para que parezcan un solo panel ajustando la alineación y el margen en Panel Settings > Geometry.

Transparencia falsa en LXTerminal

La ultima versión de la librería de widgets de VTE requiere un gestor de ventanas compositor para el fondo transparente. La versión no mantenida de VTE GTK2 tiene transparencia falsa, donde la imagen de fondo de pantalla se muestra a través de la terminal. Si prefiere la transparencia falsa, puede instalar la versión GTK 2 de LXTerminal con el paquete lxterminal-gtk2AUR.

No se le puede cambiar el tema a LibreOffice

LXDE cambia la variable de entorno SAL_USE_VCLPLUGIN que es usada para establecer el tema de LibreOffice en /usr/bin/startlxde. Cambie esa linea en el archivo para cambiar el tema. Reporte del bug.
