Xfce es un entorno de escritorio basado en GTK. Para proporcionar una experiencia de usuario completa, incluye un gestor de ventanas, un administrador de archivos, un escritorio y un panel.

Installación

Instala el grupo xfce4. También podrás querer instalar el grupo xfce4-goodies que incluye plugins adicionales y varias herramientas útiles como el editor mousepad. Xfce incluye el gestor de ventanas Xfwm por defecto.

Inicio

Elige Sesión de Xfce desde el menú de tu gestor de ventanas de tu elección, o agrega exec startxfce4 a tu Xinitrc.

Note No llames al ejecutable xfce4-session directamente; startxfce4 es el comando correcto que, a su vez, llama al anterior cuando es apropiado.

Configuración

Xfce guarda sus opciones de configuración en Xfconf. Hay varias formas de modificar estas opciones:

  • En el menú principal, selecciona Configuración y la categorá que quieras personalizar. Las categorás son programas usualmente localizados en /usr/bin/xfce4-* y /usr/bin/xfdesktop-settings.
  • xfce4-settings-editor puede ver y modificar todas las configuraciones. Las opciones modificadas aquí surtirán efecto inmediatamente. Use xfconf-query para cambiar configuraciones desde la línea de comandos; vea la documentación para más detalles.
  • Las configuraciones están alojadas en archivos XML en ~/.config/xfce4/xfconf/xfce-perchannel-xml/ que pueden ser editadas a mano. Sin embargo, los cambios realizados aquí no surten efecto inmediatamente.

Menú

Véase Xdg-menu para más información usando el sistema de menú de Free Desktop.

Menú Whisker

xfce4-whiskermenu-plugin (también parte de xfce4-goodies) es un lanzador de aplicaciones alternativo. Muestra una lista de favoritos, navega a través de todas las aplicaciones instaladas a través de botones de categorás, y soporta búsqueda difusa. Después de instalar el paquete, puede instalar el Menú de aplicaciones como primer elemento en el Panel 1 (en Configuración > Panel > Elementos añade Menú Whisker).

Configura una combinación de teclas para lanzar el menú whisker

Para configurar una combinación de teclas para lanzar el menú whisker, ve a Configuración > Teclado y luego a la pestaña Atajos de aplicación. Haz clic en el botón de Añadir, configura el comando xfce4-popup-whiskermenu y asigna la combinación de teclas deseada.

Editar entradas

Existen diversas herramientas gráficas para esta tarea:

  • MenuLibre — Un editor de menú avanzado que proporciona características modernas en una interfaz limpia y fácil de usar.
https://github.com/bluesabre/menulibre || menulibreAUR.
  • Alacarte — Editor de menú para GNOME
https://www.gnome.org/ || alacarte
  • XAME (XFCE Applications Menu Editor) — Herramienta gráfica escrita en Gambas diseñado específicamente para editar entradas de menú en Xfce, no funcionará en otros entornos de escritorio (Descontinuado)
https://redsquirrel87.altervista.org/doku.php/xfce-applications-menu-editor || xameAUR

Como alternativa, crea manualmente el archivo ~/.config/menus/xfce-applications.menu Ve el ejemplo de configuración abajo: 

<!DOCTYPE Menu PUBLIC "-//freedesktop//DTD Menu 1.0//EN"
  "http://www.freedesktop.org/standards/menu-spec/1.0/menu.dtd">

<Menu>
    <Name>Xfce</Name>
    <MergeFile type="parent">/etc/xdg/menus/xfce-applications.menu</MergeFile>

    <Exclude>
        <Filename>xfce4-run.desktop</Filename>
        <Filename>exo-terminal-emulator.desktop</Filename>
        <Filename>exo-file-manager.desktop</Filename>
        <Filename>exo-mail-reader.desktop</Filename>
        <Filename>exo-web-browser.desktop</Filename>
        <Filename>xfce4-about.desktop</Filename>
        <Filename>xfhelp4.desktop</Filename>
    </Exclude>

    <Layout>
        <Merge type="all"/>
        <Separator/>
        <Menuname>Settings</Menuname>
        <Separator/>
        <Filename>xfce4-session-logout.desktop</Filename>
    </Layout>
</Menu>

La etiqueta <MergeFile> incluye el menú por defecto de Xfce.

La etiqueta <Exclude> incluye aplicaciones que no queremos que aparezcan en el menú. En este ejemplo excluimos los atajos por defecto de Xfce, pero puedes excluir firefox.desktop o cualquier otra aplicación.

La etiqueta <Layout> define la distribución del menú. Las aplicaciones pueden ser organizadas en carpetas o como deseés. Para más detalles, véase la wiki de XFCE.

También puedes hacer cambios al menú Xfce editando los archivos .desktop como tal. Para esconder entradas, véase Desktop entries (Español)#Esconder entradas de escritorio. Puedes editar la categorá de la aplicación modificando la linea de Categories= en la entrada de escritorio, véase Desktop entries (Español)#Ejemplo de un archivo.

Configurar aplicaciones predeterminadas

Para cambiar las aplicaciones por defecto utilizadas para abrir ciertos recursos, utilice los ajustes de Aplicaciones predeterminadas. Esto cambiará el comportamiento de exo-open, que es invocado por el Iniciador de recursos como xdg-open

Escritorio

Eliminar iconos de escritorio

Ejecuta el siguiente comando: 

$ xfconf-query -c xfce4-desktop -v --create -p /desktop-icons/style -t int -s 0

Para restablecer los iconos del escritorio, ejecuta el mismo comando con un valor de 2.

Atajo para finalizar ventana

Xfce no tiene un atajo para finalizar una ventana, por ejemplo cuando un programa se congela.

Con xorg-xkill, usa xkill para finalizar una ventana de manera interactiva. Para la ventana actualmente activa, usa xdotool: 

$ xdotool getwindowfocus windowkill

O alternativamente: 

$ sh -c "xkill -id $(xprop -root -notype | sed -n '/^_NET_ACTIVE_WINDOW/ s/^.*# *\|\,.*$//g p')"

Para agregar una combinación de teclas, usa Configuración > Teclado o una aplicación como xbindkeys.

Sesión

Arranque automático

Para lanzar aplicaciones personalizadas cuando inicie Xfce, haz clic en el Menú de Aplicaciones > Configuración > Gestor de configuración y luego elige la opción de Sesión e inicio y haz clic en la pestaña de Inicio automático de aplicaciones. Verás una lista de programas que se ejecutan al iniciar sesión. Para añadir una entrada, haz clic en el botón de Añadir y rellena el formulario, especificando la ruta del ejecutable que quieras iniciar.

Las rutas de ubicaciones para inicio automático de aplicaciones se describen en la especificación de arranque automático de XDG

De forma alternativa, añade el comando que quieras ejecutar (incluyendo la configuración de variables de entorno) a xinitrc (o xprofile cuando un gestor de ventanas sea usado).

Tip A veces demorar el inicio de una aplicación puede resultar útil. Nota que especificar un comando como sleep 3 && comando en Aplicación > Arranque automático no funciona; una alternativa es utilizar la sintaxis sh -c "sleep 3 && comando"

Bloqueo de pantalla

xflock4 es el script Shell de referencia que se utiliza para bloquear una sesión de Xfce.

Trata de bloquear la pantalla con estos bloqueadores de pantalla en el orden especificado:

  • xfce4-screensaver (también parte de xfce4-goodies)
  • cinnamon-screensaver
  • mate-screensaver
  • Bloqueadores de pantalla proporcionando el método D-Bus org.freedesktop.ScreenSaver.Lock (por ejemplo light-locker)
  • xdg-screensaver(1) de xdg-utils
  • xscreensaver

Finaliza con código de retorno 1 si falla al encontrar alguno de estos.

La Lista de aplicaciones/Seguridad#Aplicaciones para bloquear la pantalla contiene una descripción corta de estos bloqueadores de pantalla junto con otras aplicaciones populares.

Para hacer que xflock4 ejecute un bloqueador de sesión personalizado, configura LockCommand en el canal de sesión de xfconf al comando a utilizar: 

$ xfconf-query --create -c xfce4-session -p /general/LockCommand -t string -s "comando-de-bloqueo"

El botón de bloqueo en el panel Botones de acción simplemente ejecuta /usr/bin/xflock4. Deber[ia funcionar como se espera mientras xflock4 funcione, es decir, uno de los bloqueadores nativos está instalado o un bloqueador personalizado está configurado para integrarse según lo propuesto arriba.

Suspensión

El que la sesión automáticamente se bloqueé o no al suspender puede configurarse a través de las propiedades de xfconf o desde la interfaz gráfica.

Para prevenir el bloqueo en la suspensión usando la linea de comandos, configura lock-screen-suspend-hibernate a false: 

$ xfconf-query -c xfce4-power-manager -p /xfce4-power-manager/lock-screen-suspend-hibernate -s false

De forma similar, configúrala a true para bloquear la sesión al suspender.

La configuración también puede ser controlada desde la interfaz gráfica: Abre la aplicación Sesión e inicio y marca o desmarca la casilla en General > Bloquear la pantalla antes de suspender.

Cuando se presiona la tecla de suspender del teclado, se puede manejar ya sea por el gestor de energá de Xfce o por systemd-logind. Para darle un precedente a logind, la siguiente configuración de xfconf debe configurarse a true: 

$ xfconf-query --create -c xfce4-power-manager -p /xfce4-power-manager/logind-handle-suspend-key -t bool -s true
Note Para comprobar cómo systemd-logind maneja los eventos cuando tiene un precedente csobre Xfce, revisa logind.conf(5)

Deshabilitar sesiones guardadas

Por usuario, las sesiones guardadas se pueden deshabilitar desmarcando Aplicaciones > Configuración > Sesión e inicio > General > Guardar sesiones automáticamente al salir o ejecutando el siguiente comando: 

$ xfconf-query --create -c xfce4-session -p /general/SaveOnExit -t bool -s false

Como alternativa, el modo kiosko de Xfce puede ser utilizado para deshabilitar el guardado de sesiones a nivel de sistema. Para deshabilitar las sesiones, crea o edita el archivo /etc/xdg/xfce4/kiosk/kioskrc y añade lo siguiente: 

[xfce4-session]
SaveSession=NONE

Podrás tener qué eliminar sesiones anteriormente guardadas. Navega a Aplicaciones > Configuración > Sesión e inicio > Sesiones guardadas y presiona el botón Eliminar sesiones guardadas, o simplemente elimina el directorio ~/.cache/sessions/.

Usar un gestor de ventanas diferente

Note Para que los cambios tengan efecto, debes eliminar las sesiones guardadas y asegurarte de que el guardado de sesiones esté deshabilitado cuando inicies sesión por primera vez. Una vez que el gestor de ventanas de tu elección esté ejecutándose, ya puedes volver a habilitar el guardado de sesiones.

Los archivos que especifican el gestor de ventanas por defecto se encuentran en las siguientes ubicaciones:

  • ~/.config/xfce4/xfconf/xfce-perchannel-xml/xfce4-session.xml - por usuario
  • /etc/xdg/xfce4/xfconf/xfce-perchannel-xml/xfce4-session.xml - en el sistema

El gestor de ventanas por defecto para el usuario puede ser configurado fácilmente usando xfconf-query: 

$ xfconf-query -c xfce4-session -p /sessions/Failsafe/Client0_Command -t string -sa xfsettingsd
$ xfconf-query -c xfce4-session -p /sessions/Failsafe/Client1_Command -t string -sa gestor_de_ventanas

Si quieres iniciar el gestor de ventanas con opciones de linea de comandos, usa estos comandos: 

$ xfconf-query -c xfce4-session -p /sessions/Failsafe/Client0_Command -t string -sa xfsettingsd
$ xfconf-query -c xfce4-session -p /sessions/Failsafe/Client1_Command -t string -s gestor_de_ventanas -t string -s --opción

Si necesitas más opciones de linea de comandos, simplemente agrega más argumentos -t string y -s --wm-option al comando.

Note Si en lugar configuras Client0_Command a nombre_de_gestor y Client1_Command a xfsettingsd, puedes prevenir que xfce4-session-logout revierta a un tema por defecto. Sin embargo, necesitarás configurar manualmente el tema de cursor.

Si quieres cambiar el gestor de ventanas por defecto a nivel de sistema, edita manualmente el archivo especificado anteriormente, cambiando xfwm4 al gestor de ventanas preferido y anadiendo más lineas <value type="string" value="--wm-option"/> para opciones adicionales de lineas de comandos si es necesario.

También puedes cambiar el gestor de ventanas añadiendo gestor_de_ventanas --replace al inicio automático utilizando la utilidad de inicio automático o ejecutando gestor_de_ventanas --replace & en una terminal y asegurándote de que la sesión se guarde al cerrar sesión. Ten en mente que este método no cambia realmente el gestor por defecto, simplemente lo reemplaza al iniciar sesión. Nota que si usas la utilidad de inicio automático, deberás deshabilitar las sesiones guardadas, ya que esto podrá hacer que el nuevo gestor de ventanas se inicie dos veces después del gestor de ventanas por defecto.

Temas

Los temas de XFCE están disponibles en xfce-look.org. Los temas de Xfwm se almacenan en /usr/share/themes/nombre_de_tema/xfwm4, y configurándolo en Configuración > Gestor de ventanas. Los temas GTK se almacenan en /usr/share/themes/nombre_de_tema/gtk-2.0 y /usr/share/themes/nombre_de_tema/gtk-3.0 y se configuran en Configuración > Apariencia.

Para lograr una apariencia uniforme para todas las aplicaciones, véase Apariencia uniforme para aplicaciones Qt y GTK.

Véase también Temas de cursor, Iconos, y configuración de tipografás.

Apariencia consistente entre ventanas SSD y CSD

Xfce actualmente usa Decoraciones de Servidor (SSD) (véase Wikipedia:Window decoration en inglés) con temas desde Xfwm para la mayorá de ventanas y Decoraciones de cliente (CSD) con temas desde sus respectivos programas para Configuración de Xfce, Imprimir, Guardar, y otros diálogos.

Los estilos de ventanas SSD de Xfwm pueden ser temificados para concordar con las ventanas CSD ajustando manualmente o creando temas en /usr/share/themes/nombre_de_tema/xfwm4 o usando una herramienta como xfwm4-theme-generatorAUR que "Crea temas xfwm4 desde decoraciones de cliente".

Revertir decoraciones de cliente

Desde Xfce 4.18, las decoraciones de cliente son opcionales y están deshabilitadas por defecto. [1]

Las aplicaciones no pertenecientes a Xfce aún pueden usar decoraciones de cliente. Para deshabilitarlas globalmente, véase GTK#Decoraciones del lado del cliente.

Sonido

Temas de sonido

Xfce soporta sonidos del sistema de freedesktop, pero no está configurado por defecto.

Para habilitar un tema de sonido:

  1. Marca la casilla Habilitar sonidos para los eventos en Configuración > Apariencia > Ajustes;
  2. En el Editor de configuración configura xsettings/Net/SoundThemeName a un tema de sonidos localizado en /usr/share/sounds/;
  3. Habilita Sonidos del sistema en el mezclador de audio (por ejemplo pavucontrol).

sound-theme-freedesktop proporciona un tema de sonidos compatible, pero carece de muchos sonidos requeridos. Una mejor elección es sound-theme-smoothAUR (SoundThemeName debe ser Smooth).

Teclas de volumen del teclado

xfce4-pulseaudio-plugin proporciona un applet de panel que tiene soporte para control de volumen desde el teclado y notificaciones de volumen. Como alternativa, puedes instalar xfce4-volumed-pulse, que también proporciona contorl de teclado y notificaciones, pero sin un icono en el panel. Esto es útil, por ejemplo, cuando se usa pasystray al mismo tiempo para un control más fino.

Como alternativa, xfce4-mixer también proporciona un applet de panel y atajos de teclado. El paquete de Arch sólo soporta ALSA, pero puedes reconstruirlo manualmente para añadir soporte de PulseAudio.

Después de instalar los applets, debes añadirlos a la barra de tareas o los atajos de teclado no funcionarán.

Para alternativas no específicas de un entorno de escritorio, véase List of applications (Español)/Multimedia (Español)#Control de volumen.

Atajos

Si no estás usando un applet o demonio que controle las teclas de volumen, puedes mapear comandos de control de volumen a tus teclas de volumen utilizando las configuraciones de teclado de Xfce. Para el sistema de sonido que estás utilizando, revisa las secciones enlazadas debajo para los comandos apropiados.

  • ALSA: véase Advanced Linux Sound Architecture#Keyboard volume control.
  • PulseAudio: véase PulseAudio#Keyboard volume control.
  • OSS: véase OSS#Keyboard volume control.

Atajos de teclado

Los atajos de teclado se definen en dos lugares: Configuración > Gestor de ventanas > Teclado y Configuración > Teclado > Combinaciones de teclas.

Agente de autenticación Polkit

The polkit-gnome agent will be installed along with xfce4-session and autostarted automatically; no user intervention is required. For more information, see Polkit#Authentication agents. El agente de autenticación polkit-gnome se instalará junto con xfce4-session e iniciado automáticamente; no se requiere intervención del usuario. Para más información, véase Polkit#Authentication agents

Un agente de autenticación de terceros para Xfce también está disponible, véase xfce-polkitAUR o xfce-polkit-gitAUR.

Suspensión de pantalla

Algunos programas usados comúnmente con Xfce controlarán las configuraciones de suspensión de pantalla y DPMS (ahorro de energía del monitor). Se discuten abajo.

Gestor de energía de Xfce

El Gestor de energía de Xfce sólo controla los ajustes de DPMS. Se pueden configurar en la interfaz de Gestor de energía dentro de la pestaña Pantalla.

Nota que cuando Gestionar energía de pantalla está desactivado, DPMS está completamente deshabilitado, pero esto no significa que el Gestor de energía simplemente dejará de controlar DPMS. Sin embargo, no controla la suspensión de la pantalla, que podría seguir habilitado incluso después de que la gestión de energía de la pantalla esté deshabilitada. [2] Para deshabilitar la suspensión de pantalla y DPMS, haz clic derecho en el icono de la bandeja de sistema del gestor de energía o clic izquierdo en el applet del panel y asegúrate de que la opción etiquetada Modo de presentación esté marcada.

XScreenSaver

Si xscreensaver está instalado y se ejecuta junto con el Gestor de energía de XFCE, podría no ser claro qué aplicación está en control de la suspensión y DPMS, ya que ambos compiten para controlar los mismos ajustes. Por lo tanto, en una situación donde es importante que el monitor no se suspenda (cuando ves un vídeo por ejemplo), se recomienda que deshabilites la suspensión y DPMS para ambas aplicaciones. Para saber más de las opciones de Xscreensaver, véase XScreenSaver#Configuración de DPMS y blanqueo de pantalla.

xset

Si ninguna de las aplicaciones anteriores está en ejecución, entonces las configuraciones de suspensión y DPMS pueden ser controladas con el comando xset, véase DPMS#Runtime settings.

Nota Hay algunos errores asociados con suspender y reactivar en algunas configuraciones. Véase [3][4].

Consejos y trucos

Soporte de montaje para Thunar y xfdesktop

Si las unidades externas conectadas no aparecen y las particiones de la instalación aparecen como dispositivos montados, en el escritorio y en Thunar, instala gvfs. Véase Udisks#Hide selected partitions para opciones de configuración más avanzadas.

Capturas de pantalla

Xfce tiene su propia herramienta de capturas de pantalla, xfce4-screenshooter. Es parte del grupo xfce4-goodies.

Atajos de teclado predeterminados: Imprimir abre el cuadro de diálogo principal, Alt+Imprimir toma una captura de la pantalla activa, Shift+Imprimir te permite seleccionar una región para capturar.

Como alternativa, se puede usar un programa independiente de captura de pantalla como scrot.

Temas o paletas de colores de terminal

Los temas o paletas de colores de la terminal se pueden cambiar en la interfaz gráfica, bajo la pestaña Colores en Preferencias. Estos son los colores que están disponibles en la mayoría de las aplicaciones de consola como Emacs, vi y otras.

Las configuraciones se guardan en Xfconf. Aunque puedes editarlas directamente, podría ser más conveniente descargar o crear un archivo de configuración de color. Los archivos por defecto están ubicados en /usr/share/xfce4/terminal/colorschemes/, los archivos personalizados pueden guardarse en ~/.local/share/xfce4/terminal/colorschemes/. Puedes seleccionar una paleta en Preferencias > Colores > Paleta de colores.

Revisa el hilo del foro Terminal Colour Scheme Screenshots (en inglés) para cientos de elecciones y temas disponibles.

Un ejemplo documentado de un archivo de paleta de colores: 

~/.local/share/xfce4/terminal/colorschemes/tango-gray.theme
[Scheme]

# Nombre de la paleta de colores
Name=Tango (gray background)

# Nombre traducido a otros idiomas
Name[ru]=Танго (серый фон)
Name[es]=Tango (fondo gris)

# Los valores de los colores son colores CSS (como #ff0000, #ffff00000000, rgb(255, 0, 0) o red)

ColorForeground=white
ColorBackground=#323232

# 16 colores separados por punto y coma:
# negro;rojo;verde;amarillo;azul;púrpura;cyan;gris claro;gris;rojo claro;verde claro;amarillo claro;azul claro;púrpura claro;cyan claro;blanco
ColorPalette=#2e3436;#cc0000;#4e9a06;#c4a000;#3465a4;#75507b;#06989a;#d3d7cf;#555753;#ef2929;#8ae234;#fce94f;#739fcf;#ad7fa8;#34e2e2;#eeeeec

# Descomenta para sobreescribir:

# TabActivityColor=#ef2929

# ColorCursor=white
# ColorCursorForeground=#323232
# ColorCursorUseDefault=FALSE

# ColorSelectionBackground=#ad7fa8
# ColorSelection=#323232
# ColorSelectionUseDefault=FALSE

# ColorBold=white
# ColorBoldUseDefault=FALSE

# "Variación del color de fondo para cada pestaña"
# ColorBackgroundVary=TRUE

# "Usa el tema de colores del sistema para el texto y el fondo"
# ColorUseTheme=TRUE

Autocompletado de env-modules en Terminal

El paquete env-modulesAUR proporciona autocompletado de consola para consolas de inicio de sesión. Sin embargo, por defecto, las sesiones de xfce4-terminal no se consideran comoLogin. PAra habilitar el autocompletado de Módulos de Entorno, marca la casilla Ejecutar comando como consola de inicio de sesión en Preferencias > General.

Gestión de color

Xfce no tiene soporte nativo para gestión de color. [5] Véase ICC profiles para alternativas.

Múltiples monitores

Xfce tiene soporte para múltiples monitores, que puede configurarse en el diálogo Aplicaciones > Configuración > Pantalla. En la pestaña Avanzado, se pueden guardar perfiles para diferentes monitores y hacer que se apliquen automáticamente en cuanto cambien los monitores conectados. Para más información, revisa el artículo de pantallas de la documentación de XFCE.

Como alternativa, se puede usar arandr para gestionar configuraciones de pantalla en forma de comandos de xrandr que pueden ser asignados a atajos de teclado de Xfce.

Agentes SSH

Por defecto, Xfce tratará de cargar gpg-agent y ssh-agent. Ya que gpg-agent está gestionado por systemd, podrías querer deshabilitarlo en los ajustes de Xfce: 

$ xfconf-query --create -c xfce4-session -p /startup/gpg-agent/enabled -t bool -s false

Si planeas usar la unidad de usuario ssh-agent.service como se describe en If you plan to use the ssh-agent.service user unit as described in Claves SSH#Iniciar ssh-agent con systemd user, deshabilita también ssh-agent en la configuración de Xfce: 

$ xfconf-query --create -c xfce4-session -p /startup/ssh-agent/enabled -t bool -s false

To use GNOME Keyring, simplemente marca la casilla Ejecutar servicios de GNOME al iniciar sesión en la pestaña Avanzado de Sesión e inicio en la configuración de XFCE. Esto también deshabilitará gpg-agent y ssh-agent.

Fuente: https://docs.xfce.org/xfce/xfce4-session/advanced#ssh_and_gpg_agents

Enrolla una ventana de fondo sin cambiar el foco a esta

Ve a Menú principal > Configuración > Ajustes de gestor de ventanas > Accesibilidad Desmarca la casilla Traer ventana al frente cuando se presione un botón del ratón 

$ xfconf-query --channel xfwm4 --property /general/raise_with_any_button --set false

Modificador de botón de ratón

Por defecto, el modificador de botón de mouse está configurado a Alt. Esto puede ser cambiado con xfconf-query. Por ejemplo, el comando puede ser configurado con la tecla Super como el modificador de botón de ratón: 

$ xfconf-query -c xfwm4 -p /general/easy_click -n -t string -s "Super"

Estrictamente hablando, usar múltiples modificadores no está soportado. Sin embargo, como alternativa, se pueden especificar múltiples modificadores si los nombres de teclas están separados con ><. Por ejemplo, para configurar Ctrl+Alt como modificadores de botón de ratón, puedes usar el siguiente comando: 

$ xfconf-query -c xfwm4 -p /general/easy_click -n -t string -s "Ctrl><Alt"

Limitar el brillo mínimo del deslizador de brillo

Limitar el brillo mínimo puede ser útil para pantallas que deshabilitan la retroiluminación con un brillo de nivel 0. En xfce4-power-manager 1.3.2 se introdujo una nueva opción escondida para configurar un valor mínimo de brillo con una propiedad de xfconf4. Añade brightness-slider-min-level como una propiedad de entero en xfconf4. Ajusta el valor entero para tener un nivel de brillo mínimo aceptable. 

$ xfconf-query --channel xfce4-power-manager --property '/xfce4-power-manager/brightness-slider-min-level' --create --type int --set 0

Añadiendo imágenes de perfil

Para agregar imágenes de perfil para cada usuario para que se muestre en el menú whisker, simplemente coloca una imagen PNG de 96x96 en /home/user/.face.

Se pueden usar programas de edición de imágenes como GIMP para escalar tus imágenes favoritas a una resolución de 96x96 y convertirlas.

Etiqueta en plugin de gestor de energía

La opción show-panel-label en xfconf de tipo int controla la etiqueta del gestor de energía, puede ser configurado para diferentes formatos de etiqueta: puede ser configurado a 0 (sin etiqueta), 1 (porcentaje), 2 (tiempo restante) o 3 (ambos).

También es accesible a travpes de la interfaz delplugin de gestor de energía en Propiedades > Mostrar etiqueta.

Usando la tecla Windows (Super) para atajos

La tecla Super es tratada como una tecla modificadora, como Ctrl y Alt, en lugar de producir una presión de tecla. Asignarle una acción prevendrá que puedas usarla para otros atajos, porque realizará esa acción además de cualquier otra que le asignes.

Para resolverlo, y hacerla más útil para atajos, instala la aplicación xcape. Esto te permite configurar las teclas modificadoras para que actúen como otras teclas cuando las presionas y liberas por sí solas.

A continuación, ve a Configuraciones > Teclado > Atajos de aplicaciones y asigna una combinación de teclas no usada, digamos, Alt+F1, al menú de aplicaciones (o cualquier acción que quieras cuando presiones la tecla Super por sí sola). Verifica que funcione.

A continuación, usa xcape para asignar Alt+F1 a la tecla Super: 

$ xcape -e 'Super_L=Alt_L|F1'

Comprueba que la tecla Super ahora realice la acción asignada a Alt+F1.

Si todo funciona, haz que sea una acción de autoarranque; ve a Configuración > Sesión e inicio > Inicio automático de aplicaciones, presiona el botón Añadir e ingresa el comando para hacer que se ejecute cada vez que inicias Xfce (si xscape ya estaba instalado, también comprueba que no haya una entrada similar registrada).

Ahora, ya puedes usar libremente la tecla Super en atajos. Por ejemplo: En Gestor de ventanas > Teclado, puedes usar Super y Arriba o Abajo para elevar o bajar ventanas.

Usar el soporte experimental de Wayland

Xfce soporta tanto labwc como wayfireAUR como compositor Wayland. Sin embargo, sólo Labwc funciona directamente; Wayfire requiere configuración adicional al archivo de sesión para hacer que funcione. Para este propósito, esta sección se enfoca en hacer que XFCE funcione cin Labwc ya que requiere el menor esfuerzo para que XFCE funcione en Wayland.

Después de instalar labwc, deberías poder cambiar a la Sesión de Xfce (Wayland) en el Gestor de ventanas de tu elección e iniciar sesión como siempre. Nota que el soporte de Wayland está marcado como experimental por una buena razón: las cosas no funcionarán como esperas y muchas cosas están generalmente rotas. Por ejemplo, los iconos de escritorio colocados por xfdesktop podrían aparecer y desaparecer conforme el escritorio gane y pierda el foco.

Usar distribuciones de teclas personalizadas con labwc

Los archivos de configuración de labwc para Xfce están ubicados en ~/.config/xfce4/labwc/ en lugar del directorio por defecto Labwc ~/.config/labwc/. Si tienes un archivo environment personalizado con distribuciones, necesitarás un archibo de bloqueo ~/.config/xfce4/labwc/lock, de otro modo la distribución se sobrescribe por startxfce4 con la distribución por defecto del sistema.

El cambio de fondo de pantalla no funciona en la configuración habitual del panel de control

El panel de control del fondo de escritorio de Xfce aún no está adaptado para funcionar con Wayland. Usa una herramienta alternativa para cambiar el fondo de pantalla como swaybg.

Solución de problemas

Modificar configuraciones no surte efecto

Si ejecutas un demonio de Xsettings, podría hacer que algunas configuraciones no surtan efecto. Deshabilítalo removiendo o comentando la linea correspondiente y reinicia Xorg.

NVIDIA y xfce4-sensors-plugin

Para detectar y usar los sensores eun una GPU Nvidia, instala libxnvctrl y después reconstruye xfce4-sensors-plugin con ABS. Otra opción es xfce4-sensors-plugin-nvidiaAUR que reemplaza xfce4-sensors-plugin.

Los applets del panel siguen alineándose a la izquierda

Añade un separador en algún punto anterior a la derecha, y habilita su propiedad Expandir.[6]

Restablecer configuraciones por defecto

Si por alguna razón necesitas volver a las configuraciones por defecto, elimina o renombra ~/.config/xfce4/: 

$ mv ~/.config/xfce4/ ~/.config/xfce4-bak

Vuelve a iniciar sesión para que los cambios surtan efecto. Si recibes el mensaje No se pudo cargar sesión segura al iniciar sesión, revisa la sección #Falla de sesión.

Falla de sesión

Los síntomas incluyen:

  • El puntero es una X y/o no aparece en absoluto;
  • Las decoraciones de ventanas han desaparecido y las ventanas no pueden ser cerradas;
  • El escritorio se torna gris y todos los iconos del escritorio desaparecen;
  • (xfwm4-settings) no inicia, reportando Estas configuraciones no pueden funcionar con tu gestor de ventanas actual (desconocido);
  • Se reportan errores por un gestor de pantallas como Ningún gestor de ventana registrado en la pantalla 0;
  • Incapaz de cargar una sesión segura:
No se pudo cargar una sesión segura.
No se pudo determinar el nombre de la sesión segura. Posibles causas: xfconfd no se está ejecutando (problema de configuración de D-Bus); variable de entorno $XDG_CONFIG_DIRS configurada incorrectamente (debe incluir "/etc"), o xfce4-session no está instalado correctamente.

Reiniciar XFCE o tu sistema podrían resolver el problema, pero una sesión corrupta también podría ser la causa. Elimina el directorio de sesiones: 

$ rm -r ~/.cache/sessions/

También asegúrate de que los directorios relevantes en $HOME sean propiedad del usuario iniciando xfce4. Véase Chown.

La barra de menú del editor Mousepad no es visible

Ejecuta el siguiente comando para hacerla visible: 

$ gsettings set org.xfce.mousepad.preferences.window menubar-visible true

Icono de papelera no es visible y applet de papelera no funciona

La papelera requiere la dependencia opcional gvfs para funcionar. Instala gvfs y reinicia el sistema.

El soporte de Wayland no funciona

Si tratas de hacer que Xfce funcione con wayfireAUR, se deben editar los archivos de sesiones para hacer que funcione.

Si no, asegúrate de tener instalado labwc.
