La precarga es la acción de colocar y mantener archivos de destino en la RAM. El beneficio es que las aplicaciones precargadas se inician más rápidamente porque la lectura desde la RAM siempre es más rápida que desde el disco duro. Sin embargo, parte de su RAM se dedicará a esta tarea, pero no más que si dejara abierta la aplicación. Por lo tanto, la precarga se usa mejor con aplicaciones grandes y de uso frecuente como Firefox y LibreOffice.

Go-preload

gopreload-gitAUR es un pequeño demonio creado en el foro de Gentoo. Para utilizarlo, primero ejecute esta orden en una terminal para cada programa que desee precargar en el arranque: 

# gopreload-prepare programa

Para los usuarios habituales, tome posesión de /usr/share/gopreload/enabled y /usr/share/gopreload/disabled: 

# chown usuario:users /usr/share/gopreload/enabled /usr/share/gopreload/disabled

Y luego gopreload para cada programa que desee precargar: 

$ gopreload-prepare programa

Posteriormente, según las instrucciones, presione Intro cuando el programa esté completamente cargado. Esto añadirá una lista de archivos que necesita el programa en /usr/share/gopreload/enabled. Para cargar todas las listas en el arranque, active gopreload.service.

Para desactivar la carga de un programa, elimine la lista apropiada en /usr/share/gopreload/enabled o muévala a /usr/share/gopreload/disabled.

Se recomienda ejecutar gopreload-prepare después de las actualizaciones del sistema para actualizar las listas de archivos. Para esta tarea, la siguiente herramienta sería útil: 

# gopreload-batch-refresh.sh

Déjelo ejecutarse sin utilizar el sistema.

Configuración

Preload

preload es un programa escrito por Behdad Esfahbod que se ejecuta como un demonio y registra estadísticas sobre el uso de programas que utilizan cadenas de Markov; Los archivos de los programas más utilizados se cargan en la memoria durante el tiempo libre de la computadora. Esto da como resultado tiempos de inicio más rápidos, ya que se necesitan recuperar menos datos del disco.

Instalación

Instale el paquete preloadAUR. Ahora puede iniciar el servicio preload y/o activarlo para que se inicie en el arranque.

Configuración

El archivo de configuración se encuentra en /etc/preload.conf, contiene configuraciones predeterminadas que deberían ser adecuadas para usuarios normales. La opción cycle le permite configurar la frecuencia de avisos al sistema de precarga para actualizar su modelo de qué aplicaciones y bibliotecas almacenar en caché.
